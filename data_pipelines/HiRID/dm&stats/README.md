# Invasive Ventilation 

Invasive ventilation (dm_vent_inv_state from _dm_ventilation.py_) column is derived by a complex heuristic which we developed for an other project. Basically, it is a voting system based on etCO2, TV, Airway code and Vent Mode. Including negative points for values incompatible with invasive ventilation. The reason is, that every measurement on it’s own is unreliable. E.g. etCo2 depends on if the Module is properly configured, airway code has to be entered manually by the bedside nurse, etc. See also example attached (vm3002 is etCO2, vm3017 the Airway code and the others mainly vent measurements – including gap of Co2 and spurious recordings, airway way longer than true). 
After the voting the periods are smoothed, and the edges are trimmed. Meaning short gaps are filled and short events deletes (there are sometime blips of etCO2 even if no patient is ventilated but module online). Consideration is given to recorded gaps in ventilation when the patient leaves the ICU for diagnostics (i.e. CT) or theater. These gaps are ignored and no new ventilation period is generated.
